# 🏛 ГИС (Государственные информационные системы)

> [!abstract] Определение
> **ГИС** — это системы, созданные на основании федеральных законов или актов госорганов для реализации их полномочий (например, Госуслуги, ЕГИСЗ в медицине).

### ⚖️ Главные правила для ГИС:
1. **Регулятор:** Только [[ФСТЭК России]] (техническая защита) и ФСБ (криптография).
2. **Основной закон:** [[ФЗ-149]] (ст. 13) и [[Приказ ФСТЭК №17]].
3. **Обязательная аттестация:** В отличие от частных систем, ГИС **нельзя** запустить без получения Аттестата соответствия.
4. **Российское ПО:** Приоритет использования отечественных средств защиты.

### 📊 Классы защищенности ГИС:

> [!abstract] Суть
> Согласно **[[Приказ ФСТЭК №17]]**, каждой государственной системе должен быть присвоен **класс защищенности**. Это определяет минимальный набор мер защиты, которые обязана внедрить организация.

### 📐 Как определяется класс?
Класс устанавливается на основе двух критериев:
1. **Уровень значимости информации (УЗ):** Насколько серьезным будет ущерб при нарушении Конфиденциальности, Целостности или Доступности (КЦД).
2. **Масштаб системы:** На каком уровне работает система (федеральный, региональный, объектный).

---

### 🏆 Таблица классов защищенности

| Масштаб системы         | УЗ 1 (Высокий - ущерб для страны\отрасли) | УЗ 2 (Средний - ущерб для региона\ведомства) | УЗ 3 (Низкий - ущерб для предприятия\объекта) |
| :---------------------- | :---------------------------------------- | :------------------------------------------- | :-------------------------------------------- |
| **Федеральный**         | **К1**                                    | **К1**                                       | **К2**                                        |
| **Региональный**        | **К1**                                    | **К2**                                       | **К3**                                        |
| **Объектный (местный)** | **К1**                                    | **К3**                                       | **К3**                                        |

* **К1 (Первый класс):** Самый высокий уровень защиты. Требуется для федеральных систем с огромным ущербом при сбое.
* **К3 (Третий класс):** Базовый уровень защиты для локальных систем.

---

### 📝 Уровни значимости информации (УЗ)
Определяются по самому высокому ущербу из трех свойств:
*   **Высокий (1):** Ущерб в масштабах всей страны/отрасли.
*   **Средний (2):** Ущерб в масштабах региона или отдельного ведомства.
*   **Низкий (3):** Ущерб в масштабах одного предприятия/объекта.

### ⚠️ Почему это важно для специалиста?
1. **Выбор СЗИ:** Если моей системе присвоен класс **К1**, я обязана использовать средства защиты с **[[Уровни доверия ФСТЭК|4-м уровнем доверия]]** и выше.
2. **Аттестация:** Чем выше класс, тем сложнее и дороже процедура **[[Аттестация объектов информатизации|аттестации]]**.
3. **Методика:** Классификация проводится **комиссией** организации и оформляется специальным **Актом классификации**.
