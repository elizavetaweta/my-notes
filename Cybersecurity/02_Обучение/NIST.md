# 🏛️ NIST (National Institute of Standards and Technology) в информационной безопасности

> [!abstract] Суть одной фразой
> NIST — Национальный институт стандартов и технологий США, нерегуляторный правительственный орган, разрабатывающий детальные руководства, фреймворки и стандарты в области кибербезопасности. Документы NIST (SP 800, FIPS) являются обязательными для федеральных органов США, но де-факто стали мировыми лучшими практиками.

В отличие от [[ISO]], стандарты NIST не сертифицируются, однако их глубина и практическая направленность делают их незаменимым инструментом для построения зрелых систем защиты.

---

## 📌 Что такое NIST и его структура

NIST — агентство Министерства торговли США, основанное в 1901 году. В области кибербезопасности ключевыми являются:

- **Computer Security Division (CSD)** — разрабатывает стандарты, руководства и метрики безопасности.
- **National Cybersecurity Center of Excellence (NCCoE)** — практические лаборатории и примеры внедрения.

Основные публикации выходят в сериях:
- **SP (Special Publication) 800** — руководства по компьютерной безопасности.
- **FIPS (Federal Information Processing Standards)** — обязательные для федеральных органов стандарты (например, FIPS 140-3 для криптографических модулей, FIPS 197 — [[AES]]).

## 🔑 Ключевые документы NIST

| Документ | Назначение |
|---|---|
| **NIST Cybersecurity Framework (CSF)** | Глобально признанный фреймворк для управления киберрисками: идентификация, защита, обнаружение, реагирование, восстановление |
| **NIST SP 800-53** | Каталог мер контроля безопасности и конфиденциальности для федеральных информационных систем (основа для многих частных компаний) |
| **NIST SP 800-171** | Защита контролируемой несекретной информации (CUI) в нефедеральных системах — обязателен для подрядчиков Минобороны США |
| **NIST SP 800-30** | Руководство по оценке рисков |
| **NIST SP 800-61** | Руководство по реагированию на инциденты |
| **NIST SP 800-63** | Цифровая идентификация (Digital Identity Guidelines) |
| **NIST SP 800-88** | Руководство по безопасному уничтожению данных (Data Sanitization) |
| **NIST SP 800-115** | Техническое руководство по тестированию на проникновение |

Кроме того, NIST стоял за стандартизацией ключевых криптографических алгоритмов:
- **[[AES]]** — Advanced Encryption Standard (FIPS 197).
- **[[SHA‑2 (SHA‑256 и SHA‑512)]]** и **[[SHA‑3 (SHA‑3‑256 и SHA‑3‑512)]]** — стандарты хеш-функций.
- **[[ECC]]** — эллиптическая криптография (FIPS 186).

## 🔗 Отличие от ISO

| Характеристика | NIST | [[ISO]] |
|---|---|---|
| **Юрисдикция** | США (национальный) | Международный |
| **Статус** | Обязателен для госорганов США; для остальных — добровольный | Добровольный, но может быть сделан обязательным через законы страны |
| **Сертификация** | Не предусмотрена | Предусмотрена (ISO 27001) |
| **Стиль** | Детальные технические руководства | Требования к системе менеджмента |
| **Примеры** | SP 800-53, CSF | ISO 27001, ISO 27701 |

> [!note] Для российского специалиста
> Прямого требования применять NIST в РФ нет, но знание его фреймворков высоко ценится на собеседованиях и при работе в международных компаниях. Многие подходы, заложенные в NIST, перекликаются с требованиями [[ФСТЭК России]] (например, каталог мер контроля SP 800-53 и требования к защите ГИС).

## 🧠 Значимость для меня как специалиста по ИБ

- Я знаю, что NIST CSF — это один из самых признанных фреймворков для управления киберрисками, и могу использовать его для оценки зрелости организации.
- Я понимаю, что многие меры из NIST SP 800-53 можно адаптировать для выполнения российских требований ([[Приказ ФСТЭК №117]], защита [[ИСПДн]]).
- Я могу ссылаться на NIST SP 800-30 при построении процесса управления рисками, который теперь необходим по Приказу ФСТЭК №117.
- Я знаю, что NIST SP 800-88 — это стандарт де-факто по безопасному уничтожению данных, и применяю его при разработке политик обращения с носителями.

## 💬 Ответ на собеседовании (кратко)

> «NIST — это национальный институт стандартов США, чьи документы (SP 800, FIPS) стали мировыми лучшими практиками. Главный фреймворк — NIST Cybersecurity Framework (CSF), который помогает управлять киберрисками. Для технических специалистов важны SP 800-53 (меры контроля), SP 800-30 (оценка рисков) и SP 800-115 (тестирование на проникновение). NIST также стандартизировал AES, SHA-2, SHA-3 и ECC».

## ❓ Самопроверка

1. **Чем NIST CSF принципиально отличается от ISO 27001?**  
   👉 CSF — это фреймворк, описывающий «что» нужно делать (функции: идентифицировать, защищать, обнаруживать, реагировать, восстанавливать). ISO 27001 — это требования к системе менеджмента, по которым можно сертифицироваться.
2. **Обязателен ли NIST для коммерческих компаний вне США?**  
   👉 Нет, но многие компании добровольно используют его как лучшие практики.
3. **Какие криптографические алгоритмы стандартизированы NIST?**  
   👉 [[AES]], [[SHA‑2 (SHA‑256 и SHA‑512)]], [[SHA‑3 (SHA‑3‑256 и SHA‑3‑512)]], [[ECC]].
