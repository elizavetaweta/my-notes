## 🔬 Методы определения

### 1. Визуальный анализ
Самый быстрый способ. Опытный глаз выхватывает характерные признаки:

| Признак | Вероятный алгоритм |
|---|---|
| 32 символа шестнадцатеричного формата (`a-f0-9`) | **MD5**, **MD4**, **NTLM** |
| 40 символов шестнадцатеричного формата | **SHA‑1** |
| 64 символа шестнадцатеричного формата | **SHA‑256** (часто) |
| 128 символов шестнадцатеричного формата | **SHA‑512** |
| Начинается с `$2a$`, `$2b$`, `$2x$`, `$2y$` | **[[Bcrypt]]** (Blowfish) |
| Начинается с `$scrypt$`, `$7$` | **[[Scrypt]]** |
| Начинается с `$argon2d$`, `$argon2i$`, `$argon2id$` | **[[Argon2]]** |
| `$1$` (префикс), затем длинная строка | **MD5‑crypt** (Linux) |
| `$5$` или `$6$` (префикс), далее длинная строка | **SHA‑256‑crypt**, **SHA‑512‑crypt** |
| 32 символа шестнадцатеричного формата, часто в верхнем регистре | **NTLM** (Windows) |
| Строка из букв, цифр, точек и слэшей длиной 13 символов | **DES‑crypt** (устаревший Unix) |
| `$y$`, `$gy$`, `$7$` и др. (зависит от версии ОС) | **Mac OS X salted SHA‑512** |

> [!note] Важный нюанс
> Сами по себе длина и алфавит не гарантируют точный алгоритм. Например, 32‑символьная hex‑строка может быть и MD5, и NTLM, и LM‑хешем. Поэтому визуальный анализ — это только фильтр, после которого применяются инструменты.

### 2. Инструменты автоматического распознавания
Если визуально не получается или нужно обработать много хешей:

- **hashid** — консольная утилита на Python, определяет тип по паттернам (поддерживает более 200 алгоритмов).
- **hash-identifier** — также Python‑скрипт, идущий в поставке с [[Kali Linux]].
- **CyberChef** — браузерный инструмент с операцией «Analyse hash».
- **Онлайн‑сервисы** (hashes.com, crackstation.net) — только для несекретных тестовых данных.

> [!example] Пример для лабораторной
> ```bash
> hashid -m '5f4dcc3b5aa765d61d8327deb882cf99'
> ```
> Вывод покажет возможные алгоритмы: MD5, MD4, NTLM, Domain Cached Credentials и т.д. Флаг `-m` выдаст соответствующий номер режима для [[Hashcat]].

## 🔗 Связь с другими понятиями

- **[[Хеш-функция]]**, **[[Хеширование]]** — базовые понятия.
- **[[Коллизия]]** — слабые алгоритмы (MD5, SHA‑1) определяются, чтобы сразу сказать «это небезопасно».
- **[[Bcrypt]]**, **[[Scrypt]]**, **[[Argon2]]** — современные алгоритмы, которые легко определить по формату.
- **[[Радужные таблицы]]** — атака, эффективная для неосоленных хешей; определение типа хеша подсказывает, есть ли соль.
- **[[Криптоанализ]]** — определение типа хеша является частью криптоанализа перехваченных или украденных данных.
- **[[John the Ripper]]**, **[[Hashcat]]** — инструменты, для которых результат определения типа хеша является входным параметром.

## 🧠 Значимость для меня как специалиста по ИБ

- Получив на пентесте или при разборе инцидента дамп хешей, я **первым делом** определяю тип. Это сразу говорит мне о культуре безопасности разработчиков.
- Я знаю, что если вижу `$2b$`, то пароли хешированы bcrypt с солью, и атака по радужным таблицам уже не пройдёт. Если MD5 без соли — я фиксирую критическую уязвимость.
- Я могу быстро оценить, сколько времени займёт перебор: MD5 — минуты на GPU, bcrypt с высоким cost factor — годы, Argon2 с большой памятью — практически нереально.
- Я понимаю, что некоторые форматы (например, `$1$` md5crypt) устарели и должны быть заменены на bcrypt или Argon2.

## 💬 Ответ на собеседовании (кратко)

> «Тип хеша определяется по длине, алфавиту и, главное, по префиксам формата. Например, 32‑символьная hex‑строка — вероятно MD5 или NTLM, префикс $2b$ — bcrypt, $argon2id$ — Argon2. Я использую утилиты hashid и hash-identifier для автоматического распознавания. Умение быстро определить тип хеша помогает сразу оценить стойкость хранения паролей и выбрать правильный режим для инструментов перебора».

## ❓ Самопроверка

1. **Какой алгоритм, скорее всего, скрывается за хешем, который начинается с `$2a$`?**
   👉 [[Bcrypt]] (Blowfish).

2. **Почему по длине 32 символа нельзя однозначно сказать, что это MD5?**
   👉 Потому что такую же длину в hex‑представлении имеют NTLM, LM‑хеш, md4, md5‑crypt и другие. Совпадение длины — необходимое, но не достаточное условие.

3. **Назовите две консольные утилиты для определения типа хеша.**
   👉 `hashid` и `hash-identifier`.

4. **Что даёт определение типа хеша специалисту при анализе утечки базы паролей?**
   👉 Моментальное понимание, насколько серьёзна утечка: если MD5 без соли — пароли восстановят за минуты, если Argon2 — система была защищена хорошо.
