# 🔄 Взаимосвязь: Угроза, Уязвимость и Риск

Для специалиста по ИБ важно понимать, как эти сущности работают в цепочке.

### 🧩 Логическая цепочка:
1. Есть **[[Угроза]]** (злодей хочет войти).
2. Он ищет **[[Уязвимость]]** (дыру в заборе).
3. Если он её находит — происходит **Инцидент**.
4. Вероятность этого инцидента и тяжесть его последствий — это и есть **[[Риск]]**.

### 💡 Пример для запоминания:

| Понятие        | Аналогия с домом                                       | Аналогия в ИТ                                               |     |
| :------------- | :----------------------------------------------------- | :---------------------------------------------------------- | --- |
| **Угроза**     | Вор хочет украсть деньги.                              | Хакер хочет украсть базу [[ПДн (Персональные данные)]]]     |     |
| **Уязвимость** | На первом этаже открыто окно.                          | У администратора стоит пароль `12345`.                      |     |
| **Риск**       | Шанс, что вор полезет именно в это окно + сумма кражи. | Вероятность подбора пароля + сумма штрафа от Роскомнадзора. |     |

### 🛡️ Как мы работаем с этим:
- Мы не можем убрать все **Угрозы** (хакеры будут всегда).
- Но мы можем закрыть **Уязвимости** (сменить пароль, поставить решетку).
- Этим мы снижаем **Риск** до приемлемого уровня.
