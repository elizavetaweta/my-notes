#term/concept #must-know #crypto


> [!abstract] Суть одной фразой
> Симметричная криптосистема — это система, в которой отправитель и получатель используют **один и тот же секретный ключ** для шифрования и расшифрования информации.

Это самый быстрый и распространённый способ обеспечения [[Конфиденциальность|конфиденциальности]] больших объёмов данных. Главная организационная сложность — как безопасно доставить этот единый ключ обеим сторонам.

---

## 📌 Как это работает

1. Алиса и Боб договариваются о **едином секретном ключе** (и должны сохранить его в тайне от всех).
2. Алиса берёт открытый текст, применяет [[Шифр|шифр]] и этот ключ — получает [[Шифртекст]].
3. Боб получает шифртекст и применяет тот же самый шифр с тем же самым ключом для расшифрования.
4. Если ключ знает кто-то третий (Ева) — вся защита рушится.

---

## 🔑 Ключевые характеристики

- **Скорость:** Очень высокая. Симметричные алгоритмы аппаратно поддерживаются процессорами (AES-NI), что позволяет шифровать гигабайты в секунду.
- **Длина ключа:** Фиксирована для каждого алгоритма (128, 192, 256 бит). Стойкость растёт экспоненциально с длиной ключа.
- **Проблема распределения ключей:** Это ахиллесова пята симметричной криптографии. Для N участников нужно N(N-1)/2 уникальных ключей, и каждый нужно безопасно доставить.

> [!note] Как решают проблему распределения
> На практике симметричные ключи почти никогда не передают «из рук в руки». Вместо этого используют [[Асимметричные криптосистемы|асимметричную криптографию]] (например, протокол Диффи-Хеллмана) для выработки общего сессионного ключа по открытому каналу, а затем уже шифруют трафик симметричным алгоритмом.

## 🧮 Основные алгоритмы

### Международные
- **[[AES]] (Advanced Encryption Standard)** — мировой стандарт, принятый NIST. Длина блока 128 бит, ключи 128 / 192 / 256 бит. Используется повсеместно: HTTPS (TLS), Wi-Fi (WPA2/WPA3), шифрование дисков (BitLocker, LUKS).
- ~~DES (Data Encryption Standard)~~ — **выведен из эксплуатации**. Ключ 56 бит, взламывается перебором.

### Российские (ГОСТ)
- **ГОСТ Р 34.12-2018** [[Кузнечик]] — современный блочный шифр. Блок 128 бит, ключ 256 бит. Аналог AES по архитектуре, но с собственной SP-сетью.
- **ГОСТ Р 34.12-2018** [[Магма]] — эволюция старого ГОСТ 28147-89. Блок 64 бита, ключ 256 бит. Всё ещё разрешён к применению.

Все государственные и значимые объекты КИИ обязаны использовать сертифицированные [[ФСБ России]] реализации этих алгоритмов.

## ⚙️ Режимы работы блочных шифров

Блочный шифр сам по себе умеет шифровать только один блок фиксированной длины. Чтобы работать с произвольными данными, применяются **режимы шифрования**:

- **ECB (Electronic Codebook)** — каждый блок шифруется независимо. ❌ **Не применять:** одинаковые блоки открытого текста дают одинаковые блоки шифртекста, что сохраняет структуру данных (видно «повторы»).
- **CBC (Cipher Block Chaining)** — каждый блок XOR-ится с предыдущим шифртекстом перед шифрованием. Нужен вектор инициализации (IV).
- **CTR (Counter)** — шифруется счётчик, результат XOR-ится с открытым текстом. Превращает блочный шифр в поточный, позволяет параллельную обработку.
- **GCM (Galois/Counter Mode)** — CTR + встроенная имитозащита (аутентификация). **Рекомендуется** для большинства современных систем, так как одновременно обеспечивает [[Конфиденциальность]] и [[Целостность]].

## 🧠 Значимость для меня как специалиста по ИБ

- Это мой **основной инструмент** для защиты хранящихся и передаваемых данных.
- При аудите конфигураций (TLS, VPN) я проверяю, какие cipher suites разрешены: наличие DES, RC4, 3DES — критическая уязвимость.
- Я обязана знать разницу между режимами: неправильный выбор (ECB вместо CBC/GCM) сводит стойкость на нет, и это часто встречается в плохо написанных приложениях.

## 💬 Ответ на собеседовании (кратко)

> «Симметричные криптосистемы используют один ключ для шифрования и расшифрования. Они очень быстрые и применяются для больших объёмов данных — например, в TLS-сессиях после рукопожатия. В РФ обязательны алгоритмы по ГОСТ — «Кузнечик» и «Магма», сертифицированные ФСБ. Главная проблема симметрии — распределение ключей, которая решается связкой с асимметричной криптографией».

## ❓ Самопроверка

1. **Почему в TLS используют сначала асимметрию, а потом симметрию?**
   👉 Асимметрия безопасно вырабатывает общий сессионный ключ по открытому каналу, а симметрия быстро шифрует основной трафик этим ключом.

2. **Чем опасен режим ECB?**
   👉 Он сохраняет паттерны открытого текста: одинаковые блоки шифруются одинаково. На зашифрованной картинке в ECB можно визуально различить контуры. Для реальных систем категорически не пригоден.

3. **Какой российский шифр пришёл на смену ГОСТ 28147-89?**
   👉 ГОСТ Р 34.12-2018 «Кузнечик» для блоков 128 бит и «Магма» как модернизированная версия старого алгоритма с блоком 64 бита.

4. **Что даёт режим GCM помимо конфиденциальности?**
   👉 Имитозащиту (аутентифицированное шифрование) — получатель может убедиться, что данные не были изменены в процессе передачи. Это закрывает и конфиденциальность, и целостность одним алгоритмом.
