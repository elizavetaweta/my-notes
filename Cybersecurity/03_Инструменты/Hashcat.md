# Hashcat: Быстрый старт на Arch/Manjaro

Краткое руководство по развертыванию и базовому использованию Hashcat в среде Manjaro Linux для задач аудита парольной безопасности.

## 1. Локация и Управление Пакетами

### Системная установка (Рекомендуется)
Установка производится через штатный менеджер пакетов. Это обеспечивает корректное сопряжение с системными драйверами OpenCL/CUDA.
```bash
sudo pacman -S hashcat wordlists
```

* **Бинарный файл:** `/usr/bin/hashcat` (автоматически доступен в `$PATH`).
* **Системные файлы и правила:** `/usr/share/hashcat/`
* **Проверка путей установки:** `pacman -Ql hashcat`

### Запуск и справка
* **Запуск в Linux:** Выполняется вводом команды `hashcat` в терминале (права root обычно не требуются, если у пользователя есть доступ к GPU).
* **Справка по работе:** Для вывода полного списка параметров, режимов и кодов хэшей используйте команду:
```bash
hashcat --help
```

### Подготовка словарей
Популярный словарь `rockyou.txt` поставляется в сжатом виде. Перед использованием его необходимо распаковать:
```bash
sudo gzip -d /usr/share/wordlists/rockyou.txt.gz
```
*Путь к словарю:* `/usr/share/wordlists/rockyou.txt`

## 2. Архитектура Команды

Базовый синтаксис:
```bash
hashcat -a [Режим_атаки] -m [Тип_хэша] [Файл_с_хэшем] [Словарь/Маска]
```

### Ключевые аргументы (`-a` и `-m`)

#### `-a` (Attack Mode) — Логика перебора
* **`-a 0` (Straight)** — Атака по словарю. Прямая сверка строк из файла.
* **`-a 3` (Mask)** — Брутфорс по заданной маске (например, только цифры заданной длины).
* **`-a 1` (Combination)** — Комбинация слов из двух разных словарей.

#### `-m` (Hash Type) — Идентификатор алгоритма
Определяет сигнатуру хэша для хэширования "на лету" и сравнения с таргетом:
* **`-m 0`** — MD5
* **`-m 1000`** — NTLM (Windows)
* **`-m 1800`** — sha512crypt (Linux `/etc/shadow`)
* **`-m 22000`** — WPA2-PSK (Wi-Fi)

*Поиск кодов:* `hashcat --help | grep -i "название_алгоритма"` (например, `grep -i "ntlm"`).

## 3. Управление в процессе работы (Runtime Keys)

При запуске перебора терминал переходит в интерактивный режим. Нажатие клавиш позволяет управлять сессией на лету:

* **`[s]` Status** — Обновить и показать текущий статус (скорость, температура GPU, прогресс, ETA).
* **`[p]` Pause** — Поставить перебор на паузу.
* **`[r]` Resume** — Возобновить перебор после паузы.
* **`[q]` Quit** — Безопасно завершить работу с сохранением текущей точки прогресса.
* **`[c]` Checkpoint** — Принудительно сохранить контрольную точку (сессию можно восстановить позже ключом `--restore`).

## 4. Практические Сценарии

### Сценарий А: Прямая атака по словарю (MD5)
```bash
hashcat -a 0 -m 0 target_hash.txt /usr/share/wordlists/rockyou.txt
```

### Сценарий Б: Атака по словарю с мутацией (Правила)
Использование правил повышает эффективность словаря, динамически изменяя регистр, добавляя спецсимволы или цифры.
```bash
hashcat -a 0 -m 1000 ntlm_hash.txt /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule
```

### Просмотр результатов (Пот-файл)
Hashcat сохраняет успешно скомпрометированные хэши в локальную базу. Повторный перебор не требуется, вывод доступен по флагу `--show`:
```bash
hashcat -m 1000 ntlm_hash.txt --show
```
