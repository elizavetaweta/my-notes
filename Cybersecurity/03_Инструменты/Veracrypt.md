# 🛡️ VeraCrypt — инструмент для шифрования данных

> [!abstract] Суть одной фразой
> VeraCrypt — это бесплатное кроссплатформенное программное обеспечение с открытым исходным кодом, предназначенное для шифрования данных «на лету». Оно позволяет создавать зашифрованные виртуальные диски, шифровать целые разделы и даже скрывать целые операционные системы. VeraCrypt — прямой наследник прекращённого проекта TrueCrypt с устранёнными уязвимостями и улучшенной криптостойкостью.

Для специалиста по ИБ VeraCrypt — не просто «программа для шифрования флешек», а мощный инструмент для защиты данных в покое (data-at-rest) с возможностью правдоподобного отрицания (plausible deniability).

---

## 📌 История и контекст

VeraCrypt был запущен 22 июня 2013 года как форк TrueCrypt после того, как в мае 2014 года разработчики TrueCrypt неожиданно объявили о прекращении поддержки проекта, заявив, что тот «содержит неисправленные уязвимости». VeraCrypt унаследовал архитектуру TrueCrypt, но существенно усилил защиту:

- **Увеличено количество итераций PBKDF2:** TrueCrypt использовал 1 000 итераций для системного раздела, VeraCrypt — от 200 000 до 500 000 итераций в зависимости от выбранной хеш-функции, что делает атаки прямым перебором паролей в 10–300 раз сложнее.
- **Добавлены современные алгоритмы:** Camellia, Кузнечик, Streebog (ГОСТ), BLAKE2s.
- **Устранены уязвимости начального загрузчика:** Добавлена поддержка SHA-256 для загрузочного раздела.
- **Внедрена поддержка EFI/UEFI:** современные BIOS теперь могут загружаться с зашифрованного раздела.

## 📌 Основные возможности

> [!note] Функциональные режимы
> VeraCrypt предоставляет несколько уровней защиты в зависимости от задачи:

- **Шифрование целых разделов и устройств (Full Disk Encryption):** шифруется весь жёсткий диск, SSD или внешний носитель. Системный раздел можно зашифровать с предзагрузочной аутентификацией (Pre-Boot Authentication).
- **Виртуальные зашифрованные диски (контейнеры):** создаётся файл-контейнер, который монтируется как обычный логический диск. Все данные внутри контейнера шифруются «на лету». Это удобно для облачных хранилищ и переноса данных.
- **Шифрование операционной системы:** вся ОС с загрузочным сектором шифруется целиком. При включении компьютера VeraCrypt Boot Loader запрашивает пароль до загрузки системы.
- **Portable Mode:** возможность запуска VeraCrypt с USB-носителя без установки в систему (требуются права администратора на целевой машине)

## 🧮 Поддерживаемые алгоритмы

Одна из сильных сторон VeraCrypt — возможность выбора и комбинирования алгоритмов.

### 🔐 Алгоритмы шифрования (блочные шифры)
Поддерживаются как одиночные шифры, так и **каскадные цепочки** (cascades), когда данные последовательно шифруются двумя или тремя разными алгоритмами с независимыми ключами:

| Алгоритм | Длина ключа | Тип |
|---|---|---|
| **[[AES]]** | 256 бит | Симметричный блочный |
| **Serpent** | 256 бит | Симметричный блочный (финалист конкурса AES) |
| **Twofish** | 256 бит | Симметричный блочный (финалист конкурса AES) |
| **Camellia** | 256 бит | Симметричный блочный (японский стандарт, одобрен ISO/IEC) |
| **[[Кузнечик]]** (Kuznyechik) | 256 бит | Российский стандарт ГОСТ Р 34.12-2018 |
| ~~[[Магма]]~~ | 256 бит | **Удалена** в версии 1.19 по результатам аудита безопасности |

> [!example] Каскадное шифрование (cascade)
> Можно выбрать, например, цепочку AES-Twofish-Serpent: каждый 128-битный блок сначала шифруется Serpent, затем Twofish, затем AES — все с независимыми 256-битными ключами. Даже если злоумышленник взломает один алгоритм, данные останутся защищены двумя другими[reference:4][reference:5].

### 🧽 Хеш-функции для формирования ключей
Для выработки ключей шифрования из пароля (Key Derivation) VeraCrypt поддерживает:

- **PBKDF2-HMAC** с выбором хеш-функции: **SHA-512**, **SHA-256**, **BLAKE2s-256**, **Whirlpool**, **Streebog (ГОСТ Р 34.11-2012)**
- **Argon2id** — современная память-жёсткая функция (аналог [[Argon2]]) с настраиваемыми параметрами памяти и времени. Использует внутренний хеш BLAKE2b

### ⚙️ Режим шифрования
VeraCrypt использует режим **XTS (XEX-based Tweaked-codebook mode with ciphertext stealing)** — стандартный режим для шифрования дисков, оптимизированный для произвольного доступа к блокам

> [!note] Почему XTS, а не GCM?
> [[GCM]] отлично подходит для потоковой передачи данных (TLS), но для шифрования дисков, где нужен произвольный доступ к любому сектору, XTS предпочтительнее. Он не требует хранения уникального nonce для каждого блока — вместо этого используется «твик» на основе номера сектора.

## 🕵️ Скрытые тома и правдоподобное отрицание (Plausible Deniability)

> [!abstract] Уникальная функция VeraCrypt
> В отличие от большинства средств шифрования (BitLocker, FileVault, LUKS), VeraCrypt поддерживает концепцию **скрытых томов (hidden volumes)**, которая обеспечивает **правдоподобное отрицание** самого факта наличия зашифрованных данных.

**Как это работает:**
1. Создаётся **внешний (outer) том** с паролем №1. В нём размещаются «отвлекающие» данные.
2. Внутри свободного пространства внешнего тома создаётся **скрытый (hidden) том** с паролем №2. Снаружи невозможно определить, есть ли скрытый том — свободное место внешнего тома и скрытый раздел выглядят одинаково (как случайные данные)
3. Если злоумышленник принуждает раскрыть пароль, вы сообщаете пароль №1, и он видит только безобидные данные. Факт существования скрытого тома доказать невозможно

**Два уровня:**
- **Скрытые тома (Hidden Volumes)** — скрытые контейнеры внутри обычных томов.
- **Скрытая операционная система (Hidden OS)** — целая ОС, работающая параллельно с «отвлекающей» ОС, но невидимая для последней

> [!danger] Важные предосторожности
> - При монтировании внешнего тома необходимо включить опцию «защитить скрытый том», иначе запись данных во внешний том может повредить скрытый.
> - Если вы не пользуетесь «отвлекающей» ОС, злоумышленник может заподозрить наличие скрытой системы. Для правдоподобия decoy-системой нужно пользоваться регулярно
> - Существуют методы обнаружения разделов со случайными данными (энтропийный анализ), но они не доказывают, что это именно VeraCrypt-том, а не просто затёртые данные

## 🔬 Аудит безопасности

VeraCrypt прошёл независимый аудит безопасности, организованный OSTIF (Open Source Technology Improvement Fund) и выполненный лабораторией QuarksLab:

- **2016–2017 годы:** аудит VeraCrypt 1.18 и его загрузчиков.
- **Результаты:** 8 критических уязвимостей, 3 средних, 15 низкой степени опасности
- **Реакция:** публикация результатов совпала с выходом VeraCrypt 1.19, в котором уязвимости были устранены. В этой же версии удалена поддержка алгоритма [[Магма]] именно по результатам аудита

С тех пор регулярных полных аудитов не публиковалось, но проект активно поддерживается, выходят обновления (текущая версия 1.26.x), а код остаётся открытым для независимого анализа.

## 🆚 Сравнение с другими решениями

| Характеристика | VeraCrypt | BitLocker (Windows) | LUKS (Linux) |
|---|---|---|---|
| **Открытый исходный код** | Да | Нет (проприетарный) | Да |
| **Кроссплатформенность** | Windows, macOS, Linux | Только Windows Pro/Enterprise | Только Linux |
| **Скрытые тома** | Да | Нет | Нет |
| **Каскадное шифрование** | Да | Нет | Нет |
| **Ключевые алгоритмы** | AES, Serpent, Twofish, Camellia, Кузнечик | AES (128/256) | Любые через dm-crypt |
| **KDF (формирование ключа)** | PBKDF2 + Argon2id | PBKDF2 (SHA-256) | Argon2 (по умолчанию в LUKS2) |
| **Простота аудита** | Высокая (~400 000 строк, но открыто) | Низкая (закрытый код) | Высокая (открытый код) |

> [!note] Когда выбрать VeraCrypt, а не BitLocker/LUKS
> VeraCrypt незаменим, когда нужна кроссплатформенность (флешка, внешний диск, которым пользуются на разных ОС), скрытые тома и правдоподобное отрицание, а также каскадное шифрование для параноидальных требований. Для простого шифрования системного диска Windows Pro достаточно BitLocker, для Linux-сервера — LUKS[reference:20].

- **[[AES]]**, **Serpent**, **Twofish**, **Camellia**, **[[Кузнечик]]** — алгоритмы шифрования, используемые VeraCrypt.
- **[[SHA‑2 (SHA‑256 и SHA‑512)]]**, **[[Стрибог]]**, **BLAKE2** — хеш-функции для PBKDF2.
- **[[Argon2]]** — функция формирования ключа (KDF) второго поколения.
- **[[Гаммирование]]**, **[[Блочный шифр]]** — режим XTS основан на блочном шифровании с твиком.
- **[[Режимы блочного шифрования]]** — сравнение XTS с GCM и другими режимами.
- **[[Ключ]]** — формирование ключей из пароля через PBKDF2/Argon2.
- **[[Соль хеша (Salt)]]** — 512-битная криптографическая соль, уникальная для каждого тома[reference:21].
- **[[Триада КЦД]]** — VeraCrypt обеспечивает конфиденциальность и целостность данных.
- **[[Правдоподобное отрицание]]** (Plausible Deniability) — уникальная возможность, реализованная через скрытые тома.

- Я знаю VeraCrypt как основной инструмент для защиты данных на конечных точках и съёмных носителях, когда штатных средств ОС недостаточно или требуется кроссплатформенность.
- Я понимаю, когда нужно применить скрытый том: при работе в юрисдикциях, где законодательство обязывает раскрывать ключи шифрования, или при перевозке конфиденциальных данных через границу.
- Я могу аргументировать выбор конкретного алгоритма: например, для российских государственных систем я включу в каскад ГОСТ-овый [[Кузнечик]] и [[Стрибог]].
- Я учитываю, что VeraCrypt уязвим к атакам на холодную память (cold boot attack), так как ключи находятся в оперативной памяти при монтированном томе, и при расследовании инцидентов это важно.
- Я знаю о результатах аудита и о том, что алгоритм [[Магма]] удалён из VeraCrypt именно по соображениям безопасности, и учитываю это при составлении рекомендаций.

## ⚖️ Применение в РФ и нормативный аспект

В контексте [[Нормативное регулирование криптографии в РФ]]:

- VeraCrypt может применяться для защиты персональных данных ([[ФЗ-152 "О персональных данных"]]), если в нём используются сертифицированные [[ФСБ России]] алгоритмы ([[Кузнечик]], [[Стрибог]]). Однако сама программа VeraCrypt не сертифицирована ФСБ как целостное средство.
- Для государственных информационных систем (ГИС) и объектов [[КИИ]] применение VeraCrypt может быть ограничено — здесь, как правило, требуются полностью сертифицированные СКЗИ (например, [[КриптоПРО]]).
- Для коммерческих организаций, не работающих с гостайной, VeraCrypt является допустимым и распространённым средством защиты данных.

## ❓ Самопроверка

1. **Чем VeraCrypt принципиально отличается от TrueCrypt?**
   👉 VeraCrypt использует в десятки и сотни раз больше итераций PBKDF2 (200 000–500 000 вместо 1 000), поддерживает современные алгоритмы (Camellia, Кузнечик, Streebog), Argon2id и исправляет уязвимости начального загрузчика.

2. **Что такое скрытый том (hidden volume) и зачем он нужен?**
   👉 Это том, расположенный внутри свободного пространства внешнего тома и неотличимый от случайных данных. Он обеспечивает правдоподобное отрицание: под принуждением вы раскрываете пароль от внешнего тома с безобидными данными, а существование скрытого тома доказать невозможно.

3. **Почему VeraCrypt использует режим XTS, а не GCM?**
   👉 XTS оптимизирован для шифрования дисков с произвольным доступом и не требует хранения уникального nonce для каждого блока. GCM предназначен для потоковой передачи, где важен сквозной контроль целостности, но для дисков он менее удобен.

4. **Можно ли применять VeraCrypt для защиты государственной тайны в РФ?**
   👉 Нет, так как VeraCrypt не сертифицирован ФСБ как целостное средство. Для гостайны требуются полностью сертифицированные СКЗИ (например, КриптоПРО).

5. **Какая уязвимость остаётся у VeraCrypt, даже если все алгоритмы стойкие?**
   👉 Атака на холодную память (cold boot attack) — при монтированном томе ключи находятся в оперативной памяти и могут быть извлечены до её очистки.
